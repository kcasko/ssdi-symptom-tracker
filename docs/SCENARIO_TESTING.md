# Scenario Testing

Use these scenarios for manual app verification.

## First Launch

- Create or select a profile.
- Confirm the dashboard opens with neutral personal-tracking language.
- Confirm navigation tabs and screen headers are readable on small screens.

## Daily Log

- Add symptoms with mixed severities.
- Add notes and save.
- Reopen the date and confirm the saved entry is visible.

## Activity Log

- Record an activity with duration, immediate impact, delayed impact, and recovery notes.
- Confirm the activity appears in summaries and reports.

## Medications And Appointments

- Add an active medication with side effects.
- Add an appointment and generate a summary.
- Confirm dates display consistently and appointment notes avoid unsupported advice.

## Reports And Exports

- Generate a health summary for a date range with data.
- Export CSV/JSON where supported by the platform.
- Confirm files use Daymark naming and neutral health-tracking copy.

## Settings

- Toggle theme settings.
- Test local backup and restore with non-production data.
